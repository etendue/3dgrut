> **任务来源**：Phase 3 测量门 [`P3.0 plan`](2026-06-09-phase3-lane-gt-method-a.md) 已立锚 —— baseline 前视 75 帧 `mean_lane_grad_corr` **0.693**（[`v3_plan_revised.md`](../../../v3_plan_revised.md) §6 Done Log，commits 247ce9c→98bd9b3）。门只测量、不改善；本 plan 是门之上的**实际改善**，分两条任务线：
> - **P3.1**（road 当 2D 纹理：沿车道线定向加密 / 平面 feature grid）——走**阶梯**：① lane-band 监督 loss + 放宽 road 几何约束（便宜先证 grad_corr 可被推动）→ ② MCMC 定向加密（gate=①见效）。平面 feature grid 留 stretch/v4。

>
> **决策依据**：2026-06-09 session 3×Explore 测绘（全部带 file:line，见 §1）+ AskUserQuestion 三决策（独立 plan 文档 / P3.1 阶梯 / P3.2 完整写+gate）。

**Goal:** 在 P3.0 门测出的 grad_corr 0.693 之上，用「最便宜先证、见效才加重」的阶梯把车道线锐度（主 KPI `mean_lane_grad_corr`）推高，且每一步对守护线（cc_psnr_masked 25.79 / lidar 22.69 / novel_lpips 0.5987）零回归、A/B 单变量可证伪。

**Architecture:** P3.1 第一刀走**训练 loss**（不动渲染架构）：把 P3.0 已有的纯张量 lane 工具（`_grad_mag`/`_luma`/`dilate_mask`/`class_mask_from_sseg`）组装成一个**可微稳定**的 lane-band 锐度 loss，train 分支前视帧加载 lane mask（P3.0 只在 val/test 加载），按现成「条件式 loss（有数据才算）」范式接进 `get_losses`；放宽 road 各向异性/scale 约束（与 V3-R1.2/R1.3 反向，A/B 权衡）让单个高斯能拉成沿线细条。第二刀走 **MCMC**：把 road 200K 粒子预算往车道线倾斜——复用 BEV 高度场把前视 lane mask 反投影成「lane-BEV 占据图」，给 road sub-strategy 的 `sample_new_gaussians(valid_indices)` 注入 near-lane 偏置（仿 `_install_perturb_mask` 的 instance-level 覆盖）。P3.2 改 bg penalty 的「杀」语义为「保留粒子 + road/actor 深度遮挡 + road 区 photometric 只路由 road 层」。

**Tech Stack:** PyTorch（纯张量 loss，复用 `F.conv2d` Sobel / `F.max_pool2d` 膨胀）、Hydra config（`layers.overrides.road.*` + `trainer.lambda_lane`）、现成 MCMC `LayeredMCMCStrategy` sub 派发、NCore aux lane zarr.itar（复用 `SsegAuxReader`）、pytest（Mac CPU 单测）、A800/inceptio 30k A/B。

## 2. 关键决策 / 契约

**D1 — lane loss 形式（避开 grad_corr 直接当 loss 的 Goodhart + 数值坑）**
- **主选（T1 实现）**：lane-band **梯度幅值 L1 匹配** `L_lane = mean_{band}|‖∇luma(pred)‖ − ‖∇luma(gt)‖|`，复用 `_grad_mag`/`_luma`/`dilate_mask`，纯可微、二阶稳定，直接推 pred 在车道线处出对的边缘强度。
- **备选（T5 红旗预案，A/B 第二档）**：lane-band **photometric reweight** —— 在 `compute_layered_l1_loss` 对 band 像素乘 `w_lane>1`（最稳、零新算子）；或调 `band_px`。
- **eval KPI `grad_corr`（Pearson）只读不动** —— loss 与 KPI 不同函数，防直接优化指标失真。

**D2 — T4 放宽几何与 V3-R1.2/R1.3 的张力（必须诚实标注）**
现状有**两道**机制特意抑制细长 road 高斯（防 novel-view hair-thin artifact）：① `anisotropy_ratio_max=8` clamp、② `lambda_road_eff_rank` eff-rank reg。P3.1 想让高斯拉成沿车道线细条 → **与之反向**。T4 = 建 P3.1 preset 放宽这两者（A/B 变量），**不改默认**；T5 A/B 必须同时看 novel_lpips（hair-artifact 代理）确认放宽没把 novel 搞坏。这是个权衡，不是纯增益。

**D3 — 定向加密接入点（T6/T7）**
改 road sub 的 `sample_new_gaussians(valid_indices)` 的 donor 集合（two-tier：near-lane 优先、不足回退全集），仿 `_install_perturb_mask` 做 road-sub instance 覆盖；**不引入新 loss、不碰 KPI、其它层字节等价**。near-lane 来自 T6 的静态 lane-BEV 占据图（训练前/周期性用前视帧累积一次，避免每步反投影开销）。

**D4 — P3.2 深度合成需补一轮 Explore（T9 第一步）**
T9 核心是把 `compute_bg_road_opacity_penalty`/`compute_bg_cuboid_opacity_penalty` 的 kill 语义改 mask-only（保留 bg 粒子、不压到 MCMC dead 阈值），改用 road 区 photometric 只路由 road 层 + 深度排序遮挡。**3dgrut 渲染 alpha 合成/深度排序的精确路径当前了解未及实现级** → T9 Step 0 先做一次 render 合成路径 Explore，再敲定「深度合成」的具体改法（不假装细节完备）。

```python
# SPDX-License-Identifier: Apache-2.0
"""Phase 3 P3.1 — lane-band sharpness loss（可微、二阶稳定）。

直接推 pred 在车道线 band 内出和 GT 一样强的边缘（Sobel 梯度幅值 L1），
是 eval KPI ``lane_grad_corr``（Pearson，只读）的**可微稳定代理**——不直接
优化 Pearson（二阶不稳 + Goodhart 自己的 KPI）。复用 per_class_eval 的纯
张量 helper（无 cv2 / NCore → Mac 可测）。契约同 ``compute_sky_loss``：无
lane / band < min_pixels → 返回 0（device/dtype 对齐、无 grad）→ trainer
可无条件调，前视-only 样本不均时不产生 NaN。
"""
from __future__ import annotations

from typing import Iterable

import torch

from threedgrut.model.per_class_eval import (
    DEFAULT_LANE_BAND_PX,
    LANE_CLASS_IDS,
    class_mask_from_sseg,
    dilate_mask,
    _grad_mag,
    _luma,
)


def compute_lane_sharpness_loss(
    rgb_pred: torch.Tensor,    # [H, W, 3] in [0, 1]（须可微）
    rgb_gt: torch.Tensor,      # [H, W, 3]
    lane_sseg: torch.Tensor,   # [H, W] lane 产物类 id
    lane_ids: Iterable[int] = LANE_CLASS_IDS,
    *,
    band_px: int = DEFAULT_LANE_BAND_PX,
    min_pixels: int = 50,
) -> torch.Tensor:
    """lane band 内亮度 Sobel 梯度幅值的 L1 匹配损失（标量）。"""
    raw = class_mask_from_sseg(lane_sseg, lane_ids)   # [H,W] bool
    band = dilate_mask(raw, band_px)                  # [H,W] bool
    bm = band.to(rgb_pred.dtype)
    n = bm.sum()
    if float(n.item()) < min_pixels:
        return torch.zeros((), device=rgb_pred.device, dtype=rgb_pred.dtype)
    gp = _grad_mag(_luma(rgb_pred.clip(0, 1)))        # 梯度流过 pred
    gg = _grad_mag(_luma(rgb_gt))
    return (torch.abs(gp - gg) * bm).sum() / (n + 1e-6)
```

> ⚠️ **实测修正（已实现 commit `0e9d7c1`）**：直接复用 `_grad_mag` 会在 pred 平坦区（gx=gy=0，训练初期 road 区常见）`sqrt(0)` **反向 NaN**（TDD `test_lane_loss_positive_and_differentiable` 抓到 `pred.grad=NaN`）。实际实现改为本模块内联 **eps-safe** `_grad_mag_safe`（`sqrt(gx²+gy²+1e-12)`）+ `import torch.nn.functional as F`，**不 import `_grad_mag`**（仍复用 `_luma`/`dilate_mask`/`class_mask_from_sseg`）；P3.0 立锚的 `_grad_mag` 保持不动（grad_corr 0.693 不受影响）。以 [`lane_loss.py`](../../../threedgrut/model/lane_loss.py) 实际代码为准。
