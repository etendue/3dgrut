### Bug #1 — ✅ 已修，未 commit

**症状**：viser 启动后浏览器一连上就崩，log 末尾：

```
File "/root/3dgrut/threedgut_tracer/tracer.py", line 489, in __create_camera_parameters

```

**Root cause**：`tracer.py:489` 是 `ValueError` 错误信息构造里的 f-string，自己 BUG（`gpu_batch.keys()` 不存在）；真正问题是它之前的所有 `if (K := gpu_batch.intrinsics) is not None` / `intrinsics_OpenCV*` 全 fall through，**gpu_batch 完全没有 intrinsics**。

### Bug #2 — ⚠️ 未修

**症状**（patched viser_gui_4d.py 加了 traceback 后捕获完整堆栈）：

```
File ".../threedgrut/layers/layered_model.py", line 409, in forward
    return self._blend_sky(outputs, gpu_batch)
File ".../threedgrut/layers/layered_model.py", line 432, in _blend_sky
    rgb_sky = sky_module(rays)
File ".../threedgrut/correction/sky_envmap.py", line 105, in forward
    h = F.relu(self.layer0(enc))
File ".../torch/nn/modules/linear.py", line 117, in forward
    return F.linear(input, self.weight, self.bias)
RuntimeError: Expected all tensors to be on the same device, but found at least two
devices, cpu and cuda:0! (when checking argument for argument mat1 in method
wrapper_CUDA_addmm)
```

**Root cause**：`SkyEnvmapMLP.layer0.weight` 在 CPU，但 `enc`（输入）在 CUDA。

往上追：
- `layered_model.py:316`：`self.layers["sky_envmap"].load_state_dict(sky_state)` 从 ckpt 加载权重。
- `sky_state` 是从 saved CPU tensors 来的 dict。`load_state_dict` 把 CPU 数据载入 `nn.Module` 时**不移动 device**。
- 训练/eval 路径 `train.py` 里 `model.cuda()` 显式调用，递归 ModuleDict 把 sky_envmap 一起移上 GPU。
- **playground engine 路径漏了**：`Engine3DGRUT.load_3dgrt_object` (`engine.py:1248-1253`) 只是 `model = LayeredGaussians(conf, specs=specs, scene_extent=scene_extent); model.init_from_checkpoint(checkpoint, setup_optimizer=False)`，没有 `.cuda()`。

**为什么 smoke 测过了？**：因为 smoke 用 identity view + ts=1000000，opacity 范围小（max 0.94 / 0.12），运气好走的是 `_single_bg_layer()` 快路径 → fast path 在 `layered_model.py:386` 直接 return，没碰 `_blend_sky`。viser 实际场景必走 multi-layer 路径，命中 `_blend_sky`。

**Day 2 推荐修法（Option A）**：在 `LayeredGaussians.init_from_checkpoint` 末尾自动 `.to(self.device)`，让所有 ModuleDict 成员（含 sky_envmap）一起搬。位置 `threedgrut/layers/layered_model.py` 第 316-317 行后面 `return` 之前。

```python
# in layered_model.py around line 316
if sky_state is not None and "sky_envmap" in self.layers:
    self.layers["sky_envmap"].load_state_dict(sky_state)
# T8.12 fix: ckpt state_dict tensors are CPU; explicitly migrate the
# whole module (incl. sky_envmap MLP, exposure model, dyn track buffers)
# to GPU so the playground engine path doesn't need to know about it.
# Trainer path also benefits — it harmlessly re-issues .cuda() later.
self.cuda()
return
```

或者更精细：只移 sky_envmap（避免误改其他 layer 行为）：

```python
self.layers["sky_envmap"] = self.layers["sky_envmap"].cuda()
```

**Day 2 备选修法（Option B）**：改 `Engine3DGRUT.load_3dgrt_object`（`engine.py:1253` 附近）加 `model.cuda()`。但这只 fix playground 入口；如果将来有别的 entry 也会重复踩。**推荐 Option A**。

## 8. Day 1 学到的坑（避免 Day 2 再踩）

1. **rsync 必须 `-e "ssh -T"`**：ssh config 里 `RequestTTY force` 会让 rsync 走 PTY，二进制数据被终端控制符吞掉。错误特征：`unexpected tag 103 (0x6e797372)` + `rsync_sender exit 12`。
2. **`pgrep -af X | grep -v pgrep`**：`pgrep -af` 自己也匹配自己（因为命令行含 X），必须过滤。
3. **vast.ai container 默认无 X11 libs**：`import cv2` 需要 `libxcb1 libgl1 libglib2.0-0 libsm6 libxext6 libxrender1`。基础镜像 `pytorch/pytorch:2.4.0-cuda12.1-cudnn9-devel` 不带。
4. **vast.ai ssh key 注入 ≠ 自动**：账号级注入用 `vastai create ssh-key`；当前实例还要 `vastai attach ssh <id> <pub>`。
5. **PyCache 干扰 patch**：sed/edit 完代码后，旧的 `__pycache__/*.pyc` 可能被 Python 优先加载（如果 source mtime ≤ pyc mtime + 编译位匹配）。重启前 `find __pycache__ -name '*.pyc' -delete` 一遍。
6. **viser 1.0.27 vs 1.0.28**：`install_env_uv.sh` 装的是 1.0.28（playground extra 默认），需要 `pip install viser==1.0.27` 降级（runbook 指定）。
7. **CUDA 12.1 vast.ai 镜像 vs 仓库限定列表**：仓库 `cuda_helper.sh` 只支持 11.8/12.4/12.6/12.8/13.0，得加 12.1 case（PyTorch cu121 wheel + kaolin cu121 wheel 都存在，patch 安全）。
8. **ssh tunnel 隐性断开**：`-o ServerAliveInterval=30` 是必须的；隧道 `RequestTTY force` + `-N -T` 组合可能不兼容 → 用 `-o ControlMaster=no -o ControlPath=none` 强制独立连接。
9. **smoke test 不代表 viser 路径**：smoke 用 identity view 可能走 `_single_bg_layer` fast path，bypass `_blend_sky` —— viser 真实场景必走 multi-layer。**Day 2 smoke3 必须用 1080×720 + 非 identity view + 至少 5 帧不同 view + 不同 ts 模拟真实使用**。
10. **当 viser print 错误信息只有一行时**：要么改 viser_gui_4d 的 except 块加 traceback（Day 1 做了），要么再单独跑 smoke3 重现拿 stack。
