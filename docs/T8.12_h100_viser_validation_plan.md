# T8.12 — vast.ai H100 验证 viser_gui_4d 完整 Gaussian 渲染

## 0. Context

Stage 8 viser_gui_4d 在 A800 上**只能跑 `--no_gaussian_render` 模式**（Ampere
datacenter 阉割了 RT cores，OptiX dlopen segfault）。这意味着浏览器看到的是
scene primitives + timeline，**没有 Gaussian 背景渲染图**。

要验证完整 4D 体验（含 Gaussian 体素渲染随 timeline 动），需要带 RT cores 的
GPU。本任务在 vast.ai H100（Hopper datacenter，第 3 代 RT cores）上端到端验证。

**前置条件**：T8.1-T8.11 已全部完成（main `e5032cf`），ckpt `/tmp/ckpt_with_viz_4d_v2.pt`
已在 A800 上 inject 好（994.8 MB，含完整 viz_4d schema）。

---

## 1. 任务目标

| # | 目标 | 验收 |
|---|---|---|
| 1 | viser_gui_4d 在 H100 上启动**不加 `--no_gaussian_render`**，不 segfault | log 显示 viser listening *:8080，无 OptiX error |
| 2 | 浏览器加载页面，Gaussian 背景渲染图正常显示 | 拖相机看见车 + 路面 + 天空，不是黑屏 |
| 3 | Timeline 拖动时 **Gaussian 背景图随 dynamic_rigids 层动**（车跟着 cuboid 走） | 视觉上看到车 + cuboid + 点云三者位置同步 |
| 4 | 帧率达标 | 1080×720 分辨率 ≥ 10 FPS（H100 单卡） |
| 5 | T8.11 per-track LiDAR 在有 Gaussian 背景下仍正常 | 点云跟车一起飘，不被 Gaussian 遮挡 |

**不在范围**：
- 不重新训练（直接用 A800 inject 好的 ckpt）
- 不评估 PSNR（这是 viz 验证不是 KPI 验证）
- 不长期租机器（一次性验证 30-60 min）

### 2.3 ssh config 加 vast.ai entry

```ssh-config
Host vast-h100
    HostName <vast.ai 给的 IP>
    User root           # vast.ai 默认 root；如改了 inceptio 镜像就 inceptio
    Port <vast.ai 给的端口>
    IdentityFile ~/.ssh/id_ed25519
    # ⚠️ vast.ai 用 tmux wrap, 非交互式命令必须 -tt

    ControlPath ~/.ssh/sock/%C
    ControlPersist 30m
```

## 3. 执行步骤（顺序）

### 3.1 租机器（5 min）

1. vast.ai 控制台筛 GPU = **H100** + DLPerf 排序，挑磁盘 ≥ 50GB 实例
2. Template 选 **PyTorch (cuDNN Runtime)** 或自定义 `nvcr.io/nvidia/pytorch:24.10-py3`
3. 启动后拿到 `ssh -p <port> root@<ip>` 信息，填进本地 ssh config
4. 测试连接：`ssh -tt vast-h100 'nvidia-smi'`，确认看到 H100

### 3.2 远端环境搭建（15 min）

```bash
ssh -tt vast-h100
# 进入后 vast.ai 默认 HOME=/root, 不需要 export HOME (跟 inceptio 镜像不同)

# 装 miniforge / conda (如果镜像没自带)
wget -q https://github.com/conda-forge/miniforge/releases/latest/download/Miniforge3-Linux-x86_64.sh
bash Miniforge3-Linux-x86_64.sh -b -p /root/miniforge3
source /root/miniforge3/etc/profile.d/conda.sh

# clone 3dgrut2 main
git clone https://github.com/<your-fork>/3dgrut2.git /root/3dgrut
cd /root/3dgrut

# 装 env (UV 路径)
bash install_env_uv.sh
source .venv/bin/activate

# 装 viser
pip install viser==1.0.27

# ⚠️ 不需要装 nvidia-ncore (NVIDIA 私有包) —— 因为我们用已 inject 的 ckpt,
# 不需要 NCoreDataset / NCore SDK
```

### 3.3 上传 ckpt（5-10 min）

```bash
# 在 Mac 上跑
rsync -avzP --inplace ~/4dviz_artifacts/ckpt_with_viz_4d_v2.pt \
    vast-h100:/root/ckpt_with_viz_4d_v2.pt

# 995 MB 上传约 80s @ 100 Mbps, 实际带宽看 vast.ai 实例
```

### 3.4 启动 viser_gui_4d（首次编译 OptiX ~5-10 min）

```bash
ssh -tt vast-h100
cd /root/3dgrut
source .venv/bin/activate

# 注意: 这次**不加** --no_gaussian_render
nohup python -m threedgrut_playground.viser_gui_4d \
    --gs_object /root/ckpt_with_viz_4d_v2.pt \
    --default_gs_config apps/ncore_3dgut_mcmc.yaml \
    --port 8080 > /tmp/viser.log 2>&1 &
disown

# 首次启动会触发 lib3dgrt_cc / lib3dgut_cc OptiX 编译 (5-10 min)
# log 关注: "Building extension module lib3dgrt_cc..." → "Loading extension..." → "Viser listening"
tail -f /tmp/viser.log
```

### 3.5 Mac 端口转发（保持窗口开）

```bash
ssh -L 8080:localhost:8080 vast-h100
# 浏览器打开 http://localhost:8080
```

## 5. 风险 & 兜底

| 风险 | 概率 | 兜底 |
|---|---|---|
| OptiX 编译失败（CUDA 版本不匹配） | 低 | 切换 vast.ai 实例镜像或 CUDA 12.4 模板 |
| 显存爆（H100 80GB 应该够） | 低 | 降低 resolution_slider 到 720p |
| viser 1.0.27 跟旧 ckpt schema 不兼容 | 已 commit 路径覆盖 | 直接 git checkout main on vast.ai |
| **NCore SDK 找不到** | 低（不需要它） | 我们用 ckpt 自带 viz_4d，不走 dataset fallback |
| ssh fail2ban（vast.ai 端） | 低 | ControlMaster 已配，避免连续重连 |
| 帧率 < 10 FPS | 可能（H100 PCIe 比 SXM5 慢） | 降低分辨率 / 关掉部分 visibility checkbox |
| 上传速度慢 | 可能 | 用 rsync `--inplace --partial` 断点续传 |

## 7. 备用方案：RTX 4090

如果 vast.ai 上 H100 缺货或太贵，**RTX 4090 24GB** 完全够用且更便宜：
- 24 GB 显存能装 1.4M 粒子（我们 ckpt 约 1.25M）
- Ada Lovelace 第 3 代 RT cores，OptiX 性能跟 H100 相当
- 价格 ¥5-8/h，省一半

切换只需把 vast.ai 控制台筛 GPU = 4090，其他步骤完全一样。
